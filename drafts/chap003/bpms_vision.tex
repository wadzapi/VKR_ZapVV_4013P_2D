На рис~\ref{bpms_vision} изображена общая схема предполагаемого взаимодействия участников виртуальной команды.
Ориентируясь на направления развития новых управленческих подходов целесообразна организация взаимодействия команды вокруг системы знаний о процессах. Предлагается организовать взаимодействие на основе знаний и документации, поставляемых совместно с моделями взаимодействия и бизнес-процесами. Таким образом, предлагается организация сообщества вокруг процессного репозитория. Реализация такого подхода к организации взаимодействия также позволяет создать систему управления знаниями о бизнес-процессах, предоставляет возможности анализа и совершенствования моделей бизнес-процессов, создания и распространения процессной документации. Опишем более подробно взаимодействие пользователей с системой при организации совместной деятельности с применением техники вариантов использования.

\textbf{Основной поток.}\\
\begin{itemize}
  \item На предварительном этапе с применением средств телекоммуникации члены команды сообща осуществляют разрабатку бизнес-процесса, обсуждают и утверждают механизмы взаимодейсвия и составляют приблизительное расписание работ. Полученные в результате данные формализуются в виде моделей бизнес-процессов и других видов моделей, регламентирующих утвержденный способ выполнения процессов, отправляются системе.
  \item Система сохраняет полученные от пользователя модели бизнес-процессов в репозитории процессов с указанием  версии документов.
  \item Каждый из участников копирует в локальный репозиторий разработанный бизнес-процесс, выделяет подпроцессы, в которые он вовлечен.
  \item Участники начинают выполнение запланированных действий, при необходимости уточняя детали через телекоммуникационные средства.
  \item Периодически участники группы соединяются с системой для проверки непротиворечивости, отправления отчета о выполнении запланированных этапов процесса.
  \item При соединении с пользователем система копирует ветвь участника команды и при отсутствии конфликтов сливает с общей ветвью бизнес-процесса.
  \item По завершению бизнес-процесса система оповещает участников, инициирует проведение общего собрания группы.
  \item На общем собрании участники, анализируя полученный опыт, предлагают идеи по рационализации и перепроектированию процесса.
\end{itemize}


\textbf{Альтернативные потоки.}\\
{\em Условия:} При выполнении работ возникли обстоятельства, требующие пересмотра и внесения изменений в процесс.
\begin{itemize}
  \item Участник команды, инициирующий изменение потока, оповещает других членов команды, участвующих в процессе.
  \item Собранием группы принимаются решения о мерах по перепроектированию и внесению изменений в процесс. Обновленный вариант бизнес-процесса отправляется в систему.
  \item Система сохраняет обновленный вариант процесса в виде новой ветви процесса в репозитории.
  \item Система анализирует данные о новой ветви процесса, при отсутствии конфликтов объединяет ветви.
  \item Система оповещает участников, которых затронули изменения об изменении процесса.
\end{itemize}

\noindent{\em Условия:} Система обнаружила несовместность новой ветви с уже существующими (сохраненными) в репозитории.
\begin{itemize}
  \item Система отменяет слияние и восстанавливает предыдущеее непротиворечивое состояние.
  \item Система оповещает инициаторов изменений и участников, интересы которых были затронуты.
  \item Общим собранием группы принимаются решения о мерах по перепроектированию и внесению изменений в процесс. Перепроектированный вариант бизнес-процесса отправляется в систему.
  \item Система сохраняет изменения внесенные в процесс в виде новой ветви.
  \item Система анализирует полученные данные о процессе, при отсутствии конфликтов объединяет новую ветви процессов.
\end{itemize}
