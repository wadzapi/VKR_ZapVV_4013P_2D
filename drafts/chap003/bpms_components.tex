
\paragraph{jBPM и Activiti.}
 При проведения исследования функциональных возможностей современных управленческих систем (см. раздел~\ref{market_research} на стр.~\pageref{market_research}) в качестве перспективных для дальнейшей доработки были выбраны СУБП с открытыми кодами jBPM и Activiti. С целью выбора одной из программ был проведен сравнительный анализ программ, учитывающий тип архитектуры, перспективы развития, технические и функциональные аспекты. 

В качестве базовой платформы для реализации СУБП нового поколения была выбрана программа Activiti. Преимуществом Activiti является поддержка открытых стандартов задания процессов (BPMN 2.0 XML), наличие необходимых функциональных компонентов и эффективность реализации архитектуры. 

Программа jBPM имеет функциональную направленность на обработку и выполнение бизнес-правил, включена в комплект процессора бизнес-правил jBoss Drools, использует проприетарный XML формат jPDL. Проект Activiti, согласно \cite{ActivitiURL}, разрабатывается командой профессиональных разработчиков компании Alfresco \cite{AlfrescoURL},  оснащен необходимым набором программных средств для решения практических задач отрасли управления бизнес-процессами. 

Продукты Activiti распространяются на условиях лицензии Apache, которая, согласно \cite{ApacheLicenseURL}, позволяет пользователю своободно использовать и изменять программное обеспечение, за исключением названия.

\paragraph{Обзор архитектуры Activiti.}
Набор программных компонентов Activiti включает компоненты для моделирования и проектирования бизнес-процессов (Activiti Modeler, Activiti Designer) и программный модуль для выполнениня бизнес-процесоов (Activiti Engine). Модуль проектирования бизнес-процессов базируется на платформе среды разработки Eclipse IDE. После провдения проектирования результат экспортируется в формат BPMN 2.0 XML. В модуле выполнения процессов реализован набор программных интерфейсов для получения и обработки BPMN XML, а также для соединения с другими необходимыми компонентами \textemdash\, системами управления базами данных, компонентами для управления, мониторинга и контроля бизнес-процессов. 


\paragraph{Компоненты репозитория процессов.}
С точки зрения организации децентрализованного взаимдействия интересна модель, принятая в системе децентрализованного  контроля версий git. Система git, согласно \cite{gitURL}, \cite{ArchitectOpenSource}, реализует функции децентрализованного контроля версий на основе концепции ветвления. В отличие от традиционных систем контроля версий в git предусмотрены возможности автоматического анализа и слияния параллельных ветвей разработки. Таким образом, возможно организовать локальное выполенение процессов с последующей синхронизацией процесса с другими участниками. При проведении синхронизации системой будет проводиться проверка и слияние параллельных ветвей выполнения процесса при отстутствии конфликтов и противоречий. Возможности сериализации и предстаавления бизнес-процессов в форме XML позволяют применить систему контроля git для контроля версий бизнес-процессов без внесения значительных доработок.


\paragraph{Телекоммуникационные компоненты}
Для организации взаимодействия виртуальной команды необходимо обеспеченить доступность телекоммуникационных средств обеспечения связи каждому члену виртуальной команды. После проведения анализа программного обеспечения телекоммуникации, учитывая специфику виртуальной организации, в качестве наиболее перспективного решения для органанизации телекоммуникаций была выбрана технология мгновенного обмена сообщениями на базе протокола XMPP.

\paragraph{Системы мгновенных сообщений.}
Организация многовенныго обмена сообщенииями позволит не только организовать взаимодействие, но и сформировать неформальные отношения между участниками виртуальной группы. Среди достоинств технологии отметим возможности организации системы с короткой обратной связью, информирование о доступности каждого из членов команды, возможности передачи документов и текстовых сообщений. Дополнительным преимуществом XMPP является доступность стандарта протокола и наличие широкого выбора программных средств с открытыми исходными кодами.

\paragraph{Система Openfire.}На основе протокола XMPP реализовано множество программных решений и компонентов мгновенного обмена сообщениями. В результате проведения анализа списка доступного обеспечения \cite{XMPPSoftwareURL}, в качестве наиболее перспективного был выделен программный комплекс OpenFire \cite{OpenFireURL}. В составе набора программных компонентов OpenFire реализованы все необходимые функции для организации взаимодействия, включая возможности обмена мгновенными сообщениями и даже возможности организации видеоконференций. Дополнительным преимуществом Openfire является возможность организации системы разграничения прав доступа на основе протокола доступа LDAP, возможности интеграции с различными системами управления базами данных. Openfire распространяется на условиях лицензии Open Source Apache, что позволяет не только свободно использовать, но и вносить изменения в программный код.
