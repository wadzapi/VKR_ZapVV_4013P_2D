\subsection{Концепции систем управления процессами}
\subsubsection{Основные понятия}
\paragraph{Управление бизнес-процессами}
Исследуя сферу управления бизнес-процессами, вопросы терминологии и общепринятые практики обратимся к руководству BPM CBOK <<Свод знаний по управлению бизнес-процессами>> \cite{AbpmpCbok2_2009}, разработанному ABPMP (Association of Business Management Professionals). 

В терминологии ABPMP CBOK \cite[с. 24]{AbpmpCbok2_2009} под {\em управлением бизнес процессами} понимают диспциплинарный подход к деятельности по идентификации, разработке, выполнению, документированию, измерению, мониторингу, и контролю бизнес-процессов (автоматизированных и неавтоматизированных) для получения устойчивых результатов, согласованных со стратегическими целями органицации.

\paragraph{Жизненный цикл управления процессами}
Жизненный цикл в практике управления бизнес-процессами, согласно ABPMP CBOK  \cite[с. 28]{AbpmpCbok2_2009}, предстваляет собой непревную циклическую последовательность из следующих деятельностей:
\begin{description}
  \item[Планирование] ---\, разработка стратегического подхода к организации процесса, задание целей и критериев оценки их достижения.
  \item[Анализ] ---\, проведение анализа текущей организации процессов на основе определенных целей и задач, оценка эффективности и влияния изменений на бизнес-среду организации.
  \item[Разработка и моделирование] ---\, разработка подхода к (ре)организации процесса, с проведением планирования ценности для потребителя. Моделирование процесса с учетом действующих в организации процессов, разработка подходов к интеграции с  компонентами организации, ее окружением. 
  \item[Реализация] ---\, внедрение процесса в структуру деятельности организации, действия по интеграции с действующими системами управления.
  \item[Мониторинг и контроль] ---\, деятельность по мониторингу выполнения процесса, оценка эффективности процесса с учетом разработанных критериев и стратегических целей.
  \item[Улучшение] ---\, выполнение новой итерации приведенных деятельностей с учетом полученных данных, пересмотр концепций и способа организации бизнес-деятельности.
\end{description}

\subsubsection{Инструменты управления процессами}

\paragraph{Фунциональная классификация}
Весь спектр программного обеспечения, предназначенного для управления предприятием, согласно книге Репина и Елиферова \cite[с. 13]{EliferovRepin_2005} можно условно разделить на следующие группы:
\begin{description}
  \item[Информационные системы] ---\, программные комплексы, направленные на сбор и обработку информации о деятельности организации, осуществление управленческого учета, предоставление руководителю инорфмации для принятия решений.  К данному классу также относятся системы, которые в литературе встречаются под названиями корпоративные информационные системы ({\em КИС}), системы планирования ресурсов предприятия ({ \em ERP-продукты}), автоматизированные рабочие места ({\em АРМ}), информационно-аналитические программные продукты и т.п. Функционал подобного класса продуктов нацелен в основном на поддержку принятия решения, имеет довольно ограниченные возможности в алгоритмической обработке полученных данных, применении ограниченного ряда эвристических референтных моделей. 
  \item[Системы моделирования бизнес-процессов] ---\, позволяют представить бизнес-процессы организации в виде модели с графической или табличной нотацией. Применяются в основном в процессах реогранизации и реинжиниринга бизнес-процессов. Применимость данных систем в процессе динамического управления предприятием довольно ограничены.
  \item[Системы класса DocFlow и Workflow] ---\, играют наибольшую роль в процессе управления, выполняют функции хранения, учета, передачи информации, обеспечивают контроль за прохождением потока документов и работ.
\end{description}
Приведенная классификация систем управления процессами по Репину выполнена с выделением классов по функциональному критерию, таким образом  характеризует состав набора целей и задач, для решения которых предназначены системы управления. 

\paragraph{Инструментарий процессного менеджера}
На основе анализа программных продуктов среди программных продуктов, ориентированных на управление бизнес-процессами, Хармон со ссылкой на BPTrends в книге \cite[с. 429]{Harmon_2007} выделил следующие:
\begin{itemize}
  \item инструменты моделирования процессов;
  \item инструменты управления бизнес-правилами; 
  \item приложения динамического управления процессами;
  \item инструменты мониторинга процессов; 
  \item приложения имитационного моделирования;
  \item инструменты моделирования организационной структуры;
  \item инструменты бизнес-анализа, хранилища бизнес-информации;
  \item наборы программных инструментов разработчика;
  \item серверы приложений;
  \item инструменты управления потоками работ;
  \item инструменты интеграции приложений предприятия;
  \item формальные языки и нотации бизнес-процессов;
  \item графические инструменты;
  \item инструменты статистического анализа;
  \item базы данных;
  \item комплекты программного обеспечения предприятия.
\end{itemize}
Приведенный Хармоном список характеризует состав программного обеспечения, используемого в деятельности по управлению и улучшению бизнес-процессов организации. В данном случае не ставилась цель выделения непересекающихся по функциональному признаку классов продуктов, напротив, Хармон выделяет связи между ними, представляя их в виде пересечения множеств на диаграмме Венна (см. Приложение~\ref{enterprise_bpms} на стр.~\pageref{enterprise_bpms}). Приведенная диаграмма позволяет сформировать общее системное видение системы управления процессами, описывает компоненты и обозначает взаимосвязи между ними.

\subsection{Автоматизация систем управления процессами}

\subsubsection{Основные понятия}
Под {\em системой управления}, согласно терминологии ITIL \cite[с. 84]{ITILGlossaryRus_2011}, понимают <<систему политик, процессов, функций, стандартов, рекомендаций и инструментов, которые обеспечивают достижение организацией (или ее частью) поставленных целей>>.

Под системой упраления бизнес-процессами {\em (СУБП)} подразумевают целый комплекс программных средств и компонентов, направленных на реализацию и поддержку деятельности по управлению бизнес-процессами на каждом из этапов жизненного цикла управления процессами. 

По определению из IT словаря крупной консалтинговой компании Gartner\cite{Gartner_bpms}, {\em cистемой управления бизнес-процессами} (в ориг. Business Process Management Suite), называют набор программного обеспечения для обеспечения и поддержки полного жизненного цикла программного обеспечения --- от выявления, формулирования, и проектирования процесса до реализации, мониторинга, анализа и последующей оптимизации. 

Схожим образом Ассоциация BPM-профессионалов \cite{abpmp_bpms} определяет систему управления бизнес-процессами {\em (СУБП)}, как <<программный комплекс, обеспечивающий моделирование, проектирование, разработку процессов и контролируемое выполнение работ и приложений. BPMS автоматически генерирует процессное приложение из процессных моделей и бизнес-правил, что позволяет осуществлять изменения очень быстро и под полным контролем.>>


\subsubsection{Функционал СУБП}
Авторами главы, посвященной вопросам моделирования и управления бизнес-процессами, книги <<Руководство по информационным системам в бизнесе>> (Handbook on Business Information Systems) \cite{ElMekawyShahzad_2010} были выделены группы функциональных возможностей, свойственных инфорационным системам управления бизнес-процессами.
Среди функций информационных систем управления бизнес-процессами на основе анализа и группировки по стадиям жизненного цикла выделяют следующие группы: \cite[с. 100 -- 101]{ElMekawyShahzad_2010}
\begin{description}
  \item[Задание процессов] ---\, функции, связанные с моделированием и разработкой бизнес-процесса.
  \item[Хранение процессов] ---\, функции обеспечения хранения и администрирования бизнес-моделей.
  \item[Изменение процессов] ---\, функции обеспечения возможностей добавления, изменения, удаления и извлечения моделей.
  \item[Межмодельные отношения] ---\, функции задания и обоснования связей между моделями процесса и бизнес-деятельностью.
  \item[Задание цели] ---\, функции целеполагания, обеспечения возможностей задания стратегической цели бизнес-деятельности.
  \item[Соотнесения целей и процессов] ---\, функции соотнесения определений процессов с решаемыми задачами и установленными целями. 
  \item[Имитационное моделирование] ---\,функции по обеспечению возможностей динамического моделирования заданных процессов.
  \item[Исполнение процессов] ---\, функции по обеспечению выполнения процессов бизнес-деятельности. 
  \item[Мониторинг и оценка производительности] ---\, функции по обеспечению возможностей мониторинга, оценки эффективности процессов.
  \item[Анализ процессов] ---\, функции по обеспечению необходимыми критериями, показателями и функциями для анализа процессов.
\end{description} 
  

\subsubsection{Шаблонные практики управления процессами}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\label{bpms_usecases}
На основе анализа 289 научных работ Аалст в статье, посвященной десятилетию концепции управления бизнеc-процессами \cite{LNCS_Aalst_2012}, формулирует и выделяет следующие 20 вариантов использования из области управления бизнес-процессами:
\begin{description}
  \item[Разработка модели](design model, DesM) ---\, проектирование модели процессов с чистого листа.
  \item[Формулировка модели на основе данных о событиях] (discover model from event data, DiscM) ---\, автоматическая генерация модели на основе записей журнала событий. 
  \item[Выбор модели из коллекции] (select model for collection, SelM) ---\, выбор модели из репозитория готовых.
  \item[Объединение моделей] (merge models, MerM) ---\, синтез новой модели на основе слияния имеющихся.
  \item[Композиция моделей] (compose models, CompM) ---\, синтез новой модели составлением из компонентов имеющихся. 
  \item[Разработка шаблонной модели](design configurable model, DesCM) ---\, проектирование семейства моделей, зависящих от набора параметров.
  \item[Слияние моделелей в шаблон](merge models into configurable model, MerCM) ---\, объединение нескольких моделей в единую шаблонную модель с параметрами.
  \item[Настройка шаблонной модели](configure configurable model, ConCM) ---\, получение модели семейства на основе задания конкретных значений параметров.
  \item[Обновление модели](refine model, RefM) ---\, внесение изменений или перепроектирование модели процесса.
  \item[Принятие модели](enact model, EnM) ---\, получение одобрения и внедрение модели в деятельность организации.
  \item[Ведение журналов событий](log event data, LogED) ---\, регистрация значимых событий бизнес-модели в файлах журнала.
  \item[Мониторинг](monitor, Mon) ---\, отслеживание прогресса в выполнении заданных моделей деятельности.
  \item[Адаптирование при выполнении](adapt while running, AdaWR) ---\, изменение параметров, подстройка модели в режиме выполнения.
  \item[Анализ производительности на модели](analyze performance based on model, PerfM) ---\, деятельность по оценке производительности процесса с применением техник имитационного моделирования.
  \item[Проверка модели](verify model, VerM) ---\, формальные процедуры по выявлению конфликтов и ошибок в модели.
  \item[Проверка адекватности модели](check conformance using event data, ConfED) ---\, оценка адекватности модели на основе критерия согласованности с данными журнала событий.  
  \item[Анализ производительности](analyze performance using event data, PerfED) ---\, оценка эффективности процессов на основе данных журнала событий.
  \item[Починка модели](repair model, RepM) ---\, внесение исправлений в модель процесса на основе анализа согласованности модели.
  \item[Расширение модели](extend model, ExtM) ---\, внесение в модель дополнительных данных, полученных из анализа событий.
  \item[Совершенствование модели](improve model, ImpM) ---\, пересмотр концепций модели с целью повышения показателей эффективности процессов.
\end{description}
Классификация деятельностей, опубликованная Аалстом, несет значительную ценность не только с теоретической, но и с практической точки зрения. Варианты использования характеризует состав деятельности, выполняемой при управлении бизнес-процессами, характеризуют состав функциональных требований к информационной системе по управлению бизнес-процессами.
