\chapter{Менеджмент проектов}


\section{}

Существуют несколько подходов к организации взаимодейсвия между участниками проекта и степени формализации поставленных задач. Так на основе степени формализованности взаимодействия между участниками в \cite[с. 34]{Diethelm_2004} выделяют следующие виды менеджмента:\\
\begin{description}
    \item[механистический менеджмент] --- выполняемые действия придерживаются строгого временного расписания и четко определенных целей; 
предполагает функционирование по принципу работающей машины.
    \item[системный (органический) менеджмент] --- цели в точности не определены и остаются открытыми, компетенция сотрудников широкого профиля, проектная команда обладает потенциалом разрешения конфликтных ситуаций;
на первое место в проекте ставятся отношения между людьми в проекте.
\end{description}


\section{Анализ риска. Методы оценки рисков}

Менеджмент риска осуществляется путем контроля проекта и контроля степени риска, охватывает следующие функции \cite[c. 91 -- 93]{Diethelm_2004}:
\begin{itemize} \itemsep0pt
  \item планирование сроков и хода выполнения работ и управление ими;
  \item планирование затрат и управление ими;
  \item анализ и оценка степени риска;
  \item рекомендация мер, которые стоит предпринять;
  \item отчетность
\end{itemize}

Оценка риска может производиться, как правило достигается за счет опроса команды экспертов, в которую входят от четырех до восьми человек. Оценка риска может производится путем оценки распределения набранных очков, выставленных экспертами для каждого из рисков.\cite[с. 102]{Diethelm_2004}\\ В качестве методов оценки рисков в \cite{Diethelm_2004} выделены:
\begin{description}
    \item[Метод линий риска] (equirisk contour method) Классификация рисков по критерию попадания в одну из зон опасности, выделенных на графике соотношения вероятности риска и стоимости корректировки проекта. 
    \item[Анализ вероятности событий] (probablistic event analysis, PEA) - учитывает взаимодействие стоимостей между отдельными элементами. Целью является составление таблиц калькуляционной профилактики и выявляение стоимостных рисков как отдельных этапов, так и проекта в целом.
    \item[ABC анализ] - выделение трех классов, посредством умножения вероятности на степень воздействия. 
\end{description}


\subsection{Методы контроля отклонений}

В качестве практических методов контроля за отклонениями фактических затрат проекта от запланированных в основном предполагают относительно простые графические изображения отклонений затрат, например в \cite[с. 330 - 336]{Diethelm_2004} предложены:
\begin{description}
  \item[Отчет о затратах и времени] - ведение анализа соотношения фактических зактрат к запланированным срокам.
  \item[Черновое исполнение и расход ресурсов] - анализ отклонений затрат и сроков на графике, с обозначением запланированных и фактических затрат.
\end{description}

Одним из наиболее важных ресурсов в задачах управления проектами является время. Соблюдение временные рамок выполнения этапов проекта, заложенных при планировании, наряду с достижением поставленных целей, рассматривается как основной показатель успешности выполнения проекта. Планирование сроков и хода выполнения работ гарантирует соблюдение сроков проекта, "носит характер расписания, на которе должна ориентроваться вся проектная деятельность".\cite[с. 340]{Diethelm_2004} На основе составленного плана сроков выполнения отдельных этапов проекта составляется план работ для участников проекта, планируется расходы для каждого из этапов проекта.


В качестве методов планирования сроков и хода выполнения работ в \cite[с. 342 -- 355]{Diethelm_2004} выделены следующие:
\begin{description}
  \item[Список сроков] -- метод, когда каждой деятельности сопоставлена продолжительность. Составление временного плана в видесписка из запланированного времени начала деятельности и времени ее окончания.
  \item[Столбчатая диаграмма] (диаграмма Ганта) -- визуальное отображение планирования в форме столбцов, отображающих продолжительность выполнения работ.  
  \item[Анализ сетевых методов] - набор методов планирования на основе анализа сетевых графиков и таблиц. Наиболее известен {\em метод критических путей} (critical path method) -- представление запланированных событий в виде узлов графика с указанием направления процессов деятельности. Для оценки продолжительности отдельных процессов применяют экспертные оценки и специальноые методы анализа, например метод PERT.  
\end{description}
Одним из основных элементов планирования временных ресурсов является оценка продолжительности каждого из элемнетов деятельности (рабочего пакета). Для оценки продолжительности выполения рабочего пакета на практике наиболее применительны методы экспетных оценок. При составлении временного плана для получения срока завршения проекта вычисляют наиболее ранний ({\em прямой расчет}) и наиболее поздний ({\em обратный расчет}) срок начала каждой из запланированных деятельностей. При проведении {\em прямого расчета} за основу принимают срок начала проекта, а сроки начала последующих деятельностей получают прибавлением временной оценки продолжительности их выполнение к сроку окончания предыдущих деятельностей. При {\em обратном расчете} за основу берут наиболее ранний срок заверщения проекта, а сроки наиболее позднего начала каждой деятельности получают путем вычитания оценок продолжитеьности выполнения каждой деятельности из сроков наиболее позднего начала предыдущих. Разница между наиболее ранним и наиболее поздним сроком начала деятельности определяет {\em резерв времени} на ее выполнение.\cite[с. 355 -- 357]{Diethelm_2004}   

В качестве основной меры отслеживания и контроля соблюдения сроков выполнения проекта в \cite[с. 357 -- 370]{Diethelm_2004} предложен анализ {\em плана ключевых событий} (milestone plan). План ключевых событий предволагает занесение в общий план и расписания участников списка заранее определенных событий, отражающих прогресс выполнения проекта. Определение размницы между запланированным и фактическим временем наступления ключевых событий проекта позволяет определить тенденции в сроках выполнения проекта {\em (метод анализа тенденций ключевых событий}). На основе МТА анализа возможно сделать оценку выполнимости поставленных сроков, оценить качество проведенного планирования.
